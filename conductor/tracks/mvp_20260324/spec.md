# Specification: MVP Link Receiver & Summarizer

## Goal
Build the core skeleton of the SecondMobileBrain application using Flutter. The focus is on learning the fundamental architecture of Flutter while implementing real capability.

## Requirements
1. **Flutter App Skeleton**: Create the initial Flutter application codebase.
2. **Phase 1: Receiving Data**: Integrate the `receive_sharing_intent` package so the app can catch URLs shared from other Android apps.
3. **Phase 2: Content Summarization**: Integrate `google_generative_ai`, `http`, and `html` packages to fetch the shared URL content and summarize it using the Gemini API.
4. **Phase 3: History & Persistence**: (Optional for MVP, but good for learning) Keep a local or Firebase-backed history of summarized links.

## Educational Focus
The developer is new to Flutter but learns fast by seeing. Each step should be demonstrated clearly with functioning code, followed by explanations of Flutter-specific concepts (like Widgets, State Management, and async operations in Dart).
